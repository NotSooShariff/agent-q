## Full Stack Best Practices

**Architecture**
- Use layered architecture (presentation, business, data)
- Implement proper API contracts
- Version APIs appropriately
- Use DTO/ViewModel patterns
- Separate concerns clearly

**Communication Between Layers**
- Define clear API contracts
- Use TypeScript for type safety across stack
- Implement proper error handling
- Use consistent data formats (JSON)
- Handle loading and error states

**Authentication & Authorization**
- Implement secure auth flows
- Use HTTP-only cookies for tokens
- Validate permissions on backend
- Handle token refresh
- Implement RBAC when needed

**Testing**
- Unit tests for business logic
- Integration tests for APIs
- E2E tests for critical flows
- Test frontend components
- Mock external dependencies

**Performance**
- Optimize database queries
- Implement caching (Redis, CDN)
- Use code splitting on frontend
- Optimize images and assets
- Monitor performance metrics

## Common Full Stack Patterns

**CRUD Operations**
- Frontend: Form component + API calls + state management
- Backend: Controller → Service → Repository → Database
- Validation on both frontend and backend

**Authentication Flow**
- Login form → API call → JWT token → Store in cookie
- Protected routes on frontend
- Auth middleware on backend

**Real-time Features**
- Use WebSockets or Server-Sent Events
- Implement proper reconnection logic
- Handle conflicts and race conditions
