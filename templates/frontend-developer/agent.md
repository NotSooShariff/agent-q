## Best Practices

**Component Architecture**
- Keep components small and focused
- Extract reusable logic into hooks
- Use composition patterns
- Avoid prop drilling (use context when needed)
- Follow single responsibility principle

**Styling**
- Use CSS modules or scoped styles
- Follow BEM or similar naming conventions
- Implement responsive breakpoints
- Use CSS variables for theming
- Optimize for performance (avoid layout thrashing)

**State Management**
- Keep state as local as possible
- Lift state only when necessary
- Use derived state when appropriate
- Avoid unnecessary re-renders
- Implement proper error boundaries

**Testing**
- Write unit tests for components
- Add integration tests for user flows
- Test accessibility with axe or similar
- Use testing library best practices
- Mock API calls appropriately

**Performance**
- Use React.memo for expensive components
- Implement useCallback and useMemo wisely
- Optimize images (WebP, lazy loading)
- Use CDN for static assets
- Monitor Core Web Vitals

## Framework-Specific Guidance

**React**
- Use functional components and hooks
- Implement proper dependency arrays
- Avoid inline function definitions in JSX
- Use Suspense for code splitting
- Follow hooks rules

**Vue**
- Use Composition API
- Implement proper reactivity
- Use computed properties for derived state
- Follow Vue style guide
- Optimize with v-memo

**Next.js/Nuxt**
- Use Server Components when appropriate
- Implement proper data fetching (SSR/SSG/ISR)
- Optimize images with next/image
- Implement proper routing
- Use middleware for auth

**Angular**
- Use reactive forms
- Implement proper change detection
- Use OnPush strategy when possible
- Follow Angular style guide
- Use dependency injection
